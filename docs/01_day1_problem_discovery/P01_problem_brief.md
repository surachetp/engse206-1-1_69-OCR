# P01 — Problem Brief

## Problem statement

ปัจจุบันพนักงานหอพักต้องเดินจดบันทึกค่ามิเตอร์ไฟฟ้าประจำเดือนด้วยการเขียนลงกระดาษและนำกลับมาคีย์ข้อมูลลงคอมพิวเตอร์ด้วยตนเอง ทำให้เกิดขั้นตอนการทำงานซ้ำซ้อน ข้อมูลไม่เป็นมาตรฐาน และมีความเสี่ยงต่อความผิดพลาดจากมนุษย์ (Human Error) เช่น การจดเลขผิด การอ่านลายมือผิด หรือการคีย์ข้อมูลสลับกัน ส่งผลให้การออกบิลค่าไฟล่าช้า เกิดความคลาดเคลื่อนของข้อมูล และอาจนำไปสู่ข้อพิพาทกับผู้เช่าห้องพักเกี่ยวกับค่าใช้จ่ายที่ไม่ถูกต้อง

## Target outcome

มีระบบ Mobile Application สำหรับพนักงานจดมิเตอร์ที่ใช้เทคโนโลยี OCR ในการสแกนและอ่านค่าจากมิเตอร์ไฟฟ้าแบบจานหมุน (Analog Meter) พร้อมระบบ Web Admin สำหรับตรวจสอบ ยืนยันข้อมูล และบริหารจัดการประวัติการอ่านมิเตอร์ เพื่อให้การออกบิลค่าไฟมีความถูกต้อง รวดเร็ว โปร่งใส และสามารถตรวจสอบย้อนหลังได้

## Business/User Goals

### Business Goals

- ลดความผิดพลาดจากการจดและคีย์ข้อมูลมิเตอร์ด้วยตนเอง
- ลดระยะเวลาในการเก็บข้อมูลและออกบิลค่าไฟประจำเดือน
- สร้างฐานข้อมูลประวัติการใช้ไฟฟ้าที่ถูกต้องและตรวจสอบย้อนหลังได้
- เพิ่มความโปร่งใสในการบริหารจัดการค่าไฟภายในหอพัก
- ลดจำนวนข้อร้องเรียนเกี่ยวกับความผิดพลาดของบิลค่าไฟ

### User Goals

- พนักงานสามารถบันทึกเลขมิเตอร์ผ่านมือถือได้อย่างรวดเร็ว
- ลดการพกเอกสารและการคีย์ข้อมูลซ้ำซ้อน
- มีระบบแจ้งเตือนเมื่อพบค่ามิเตอร์ผิดปกติ
- ฝ่ายบัญชีสามารถตรวจสอบข้อมูลและออกบิลได้ง่ายขึ้น
- เจ้าของหอพักสามารถดูข้อมูลและรูปภาพหลักฐานย้อนหลังได้
- ผู้เช่าได้รับบิลค่าไฟที่ถูกต้องและตรวจสอบได้

## Primary Users

| User | Goal | Pain point |
|---|---|---|
| Meter Reader (พนักงานจดมิเตอร์) | บันทึกเลขมิเตอร์ได้รวดเร็ว ถูกต้อง และส่งข้อมูลเข้าสู่ระบบได้ทันที | แสงสะท้อนจากฝาครอบมิเตอร์, ตัวเลขจานหมุนอ่านยาก, ต้องจดและคีย์ข้อมูลซ้ำ |
| Billing Officer (ฝ่ายบัญชี/ออกบิล) | ตรวจสอบข้อมูลและออกบิลค่าไฟได้ตรงเวลา | ข้อมูลล่าช้า, ตัวเลขผิดพลาดจากการจดหรือคีย์ข้อมูล |
| Building Owner (เจ้าของหอพัก) | ติดตามรายได้ค่าไฟและตรวจสอบข้อมูลย้อนหลัง | ไม่มีหลักฐานยืนยันเมื่อเกิดข้อร้องเรียนจากผู้เช่า |
| Tenant (ผู้เช่าห้องพัก) | ได้รับบิลค่าไฟที่ถูกต้องและโปร่งใส | ไม่สามารถตรวจสอบที่มาของตัวเลขในบิลได้ |
| System Administrator | ดูแลระบบและความปลอดภัยของข้อมูล | ต้องรักษาความถูกต้องของข้อมูลและสิทธิ์การเข้าถึง |

## Success Indicators

- พนักงานสามารถสแกนและยืนยันค่ามิเตอร์ได้ภายใน 3 คลิกต่อห้อง
- ระบบ OCR อ่านค่ามิเตอร์ได้ถูกต้องไม่น้อยกว่า 95%
- ระยะเวลาการเก็บข้อมูลและเตรียมออกบิลลดลงอย่างน้อย 50%
- ข้อมูลมิเตอร์และรูปภาพถูกจัดเก็บในระบบกลางและค้นหาได้
- สามารถตรวจสอบรูปภาพหลักฐานย้อนหลังได้ทุกห้อง
- จำนวนข้อร้องเรียนเกี่ยวกับการคำนวณค่าไฟลดลงอย่างมีนัยสำคัญ
- ฝ่ายบัญชีสามารถตรวจสอบและอนุมัติข้อมูลได้จากระบบเดียว

## Assumptions

| Assumption | Impact if wrong | How to verify |
|---|---|---|
| พนักงานสามารถใช้สมาร์ทโฟนถ่ายภาพมิเตอร์ได้อย่างมีประสิทธิภาพ | ภาพไม่มีคุณภาพเพียงพอสำหรับ OCR | ทดลองใช้งานจริงกับมือถือของพนักงาน |
| การถ่ายภาพมุมเอียง 15–30 องศาช่วยลดแสงสะท้อนได้ | OCR อ่านค่าไม่ได้จากภาพสะท้อน | ทดสอบภาคสนามกับมิเตอร์จริงหลายสภาพแสง |
| พื้นที่ติดตั้งมิเตอร์มีสัญญาณอินเทอร์เน็ตเพียงพอ | ไม่สามารถส่งข้อมูลขึ้นระบบได้ทันที | สำรวจสัญญาณอินเทอร์เน็ตในพื้นที่ใช้งานจริง |
| ระบบ OCR สามารถอ่านตัวเลขจากมิเตอร์จานหมุนได้แม่นยำ | ต้องแก้ไขข้อมูลด้วยมือจำนวนมาก | ทดสอบโมเดล OCR กับชุดข้อมูลจริง |
| พนักงานยอมรับการเปลี่ยนแปลงจากกระดาษมาใช้ระบบดิจิทัล | ระบบอาจไม่ได้รับการใช้งานจริง | สัมภาษณ์และทดลองใช้งานกับผู้ใช้จริง |
| ข้อมูลการใช้ไฟฟ้าสามารถจัดเก็บในระบบได้ตามนโยบายองค์กร | อาจมีข้อจำกัดด้านกฎหมายหรือความเป็นส่วนตัว | ตรวจสอบนโยบายข้อมูลและข้อกำหนดขององค์กร |
| ฝ่ายบัญชีพร้อมใช้งานระบบ Web Admin ในกระบวนการออกบิล | ต้องกลับไปใช้วิธีเดิมบางส่วน | ทดสอบการใช้งานและเก็บ Feedback จากผู้ใช้งาน |
