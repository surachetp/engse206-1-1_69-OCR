P02 — Stakeholder, Context and Scope			
			
Stakeholder Map			
			
Stakeholder	Role / interest	Goal	Concern / conflict
Meter Reader (พนักงานจดมิเตอร์)	ผู้เดินบันทึกข้อมูลดัชนีมิเตอร์ไฟฟ้าตามห้องพักหน้างาน	สแกนและบันทึกเลขมิเตอร์ได้รวดเร็ว ถูกต้อง โดยไม่ต้องพกกระดาษปากกาหรือพิมพ์เอง	แสงสะท้อนจากฝาครอบบังตัวเลข, สัญญาณเน็ตอับตามซอกตึก, ตัวเลขจานหมุนก้ำกึ่งเลื่อนหลักอ่านยาก
Billing Officer (ฝ่ายบัญชี/ออกบิล)	ผู้ตรวจสอบตัวเลขและคำนวณยอดเพื่อออกบิลค่าใช้จ่ายประจำเดือน	ได้รับข้อมูลตัวเลขมิเตอร์ที่แม่นยำ 100% เพื่อออกบิลได้ตรงเวลา	พนักงานจดเลขผิดหรือคีย์สลับจนออกบิลพลาด ทำให้เสียเวลาทำเรื่องยกเลิกและออกบิลใหม่
Building Owner (เจ้าของหอพัก)	ผู้ควบคุม ติดตาม และตรวจสอบรายได้กับประสิทธิภาพพลังงานภาพรวม	ติดตามภาพรวมรายได้ค่าไฟ และมีหลักฐานรูปถ่ายหน้าปัดย้อนหลังไว้ตรวจสอบ	เกิดข้อพิพาทหรือโดนผู้เช่าร้องเรียนว่าค่าไฟแพงเกินจริงโดยไม่มีหลักฐานชัดเจน, ข้อมูลตกหล่นทุจริต
Tenant (ผู้เช่าห้องพัก)	ผู้ใช้งานไฟฟ้าและผู้รับภาระค่าใช้จ่ายตามบิลประจำเดือน	ได้รับบิลค่าไฟที่ถูกต้องตรงตามการใช้งานจริง มีรายละเอียดดัชนีมิเตอร์ชัดเจน	บิลค่าไฟแพงเกินจริงจากระบบหรือคนจดผิด, ขาดความโปร่งใสในการเรียกเก็บเงิน
System Administrator	ผู้ดูแลเสถียรภาพระบบ สิทธิ์ผู้ใช้งาน และความปลอดภัยฐานข้อมูล	ระบบหลังบ้านและโครงสร้างฐานข้อมูลทำงานได้ต่อเนื่อง ปลอดภัย	ความถูกต้องของข้อมูลดิบจาก OCR และการจัดการสิทธิ์การเข้าถึงข้อมูลส่วนบุคคลของผู้เช่า

Scope			
			
In scope			
			
Mobile Camera UI (Anti-Reflection Guide): หน้าต่างกล้องบนแอปพร้อมกรอบ Bounding Box และเซ็นเซอร์ช่วยบอกให้พนักงานเอียงกล้อง 15-30 องศาเพื่อหลบแสงสะท้อน			
			
Image Pre-processing Engine: โค้ดปรับแต่งภาพถ่าย (เช่น Perspective Transform) ดึงภาพที่ถ่ายมุมเอียงให้แบนราบและตรง เพื่อให้โมเดล AI อ่านได้ง่ายขึ้น			
			
OCR Meter Reader Model: โมเดลประมวลผลอ่านตัวเลข 5 หลักจากมิเตอร์แบบจานหมุน (Analog)			
			
Local Validation Logic: ระบบคำนวณและแจ้งเตือนความผิดปกติทันทีที่หน้างาน (เช่น ค่าใหม่น้อยกว่าเดือนก่อน หรือหน่วยกระโดดเกินจริง)			
			
Offline Data Storage: ระบบบันทึกข้อมูลและรูปถ่ายลงในเครื่องชั่วคราวเมื่อไม่มีสัญญาณเน็ต และจะซิงค์ขึ้น Cloud อัตโนมัติเมื่อต่อ Wi-Fi			
			
Web Admin Portal: หน้าเว็บหลังบ้านสำหรับจัดการห้องพัก, ตรวจดูรูปถ่ายหน้าปัดย้อนหลัง, และกดออกรายงานสรุปยอดค่าไฟ			
			
Out of scope			
			
Paper Printing System: ระบบพิมพ์บิลหรือใบเสร็จรับเงินออกมาเป็นกระดาษโดยตรง (ส่งต่อข้อมูลให้ระบบภายนอกทำแทน)			
			
Online Payment Gateway: ระบบรับชำระเงินออนไลน์, การผูกบัญชีธนาคาร หรือการตัดบัตรเครดิต			
			
Direct Tenant Notification: ระบบส่ง SMS หรือ Line แจ้งเตือนบิลค่าไฟตรงหาผู้เช่า (สิ้นสุดกระบวนการที่การส่งยอดให้ฝ่ายบัญชี)			
			
Smart Meter Integration: การเชื่อมต่อระบบกับมิเตอร์ดิจิทัลอัจฉริยะ (IoT) ที่ส่งข้อมูลผ่านโครงข่ายไฟฟ้าโดยตรง (ระบบนี้โฟกัสเฉพาะมิเตอร์จานหมุนแบบเดิม)			
			
Constraints and Ethics/Privacy			
			
Constraint / issue	Impact	Response	
ฝาครอบพลาสติกใสสะท้อนแสงแฟลช	ภาพเกิดจุดบอดสีขาวสะท้อนเลนส์กล้อง ทำให้โมเดล OCR อ่านค่าตัวเลขไม่ติดหรืออ่านผิด	ออกแบบ UI บังคับให้ถือกล้องเอียง 15-30 องศา และใช้เทคนิค OpenCV ทำ Perspective Transform ปรับภาพให้ตรงก่อนสแกน	
พื้นที่ทางเดิน/ซอกตู้มิเตอร์อับสัญญาณเน็ต	แอปพลิเคชันค้าง ไม่สามารถส่งรูปไปประมวลผลบน Cloud ระหว่างพนักงานเดินตรวจได้	พัฒนาสถาปัตยกรรมแบบ Offline-first ให้บันทึกรูปและใช้ Logic ตรวจสอบเบื้องต้นในเครื่องก่อน รอซิงค์ตอนต่อ Wi-Fi	
ความก้ำกึ่งของมิเตอร์จานหมุนช่วงเปลี่ยนหลัก	เฟืองตัวเลขกำลังเลื่อนครึ่ง ๆ กลาง ๆ อาจทำให้ AI สับสนและระบุตัวเลขผิดพลาด	กำหนดเป็นเกณฑ์ Human-in-the-loop: ให้มนุษย์ (พนักงาน) ตรวจทานและกดยืนยัน/แก้ไขขั้นสุดท้ายบนจอเสมอ ห้ามระบบบันทึกออโต้ทันที	
ความเสื่อมสภาพของอุปกรณ์หน้างาน	มิเตอร์เก่ามาก ตัวเลขจาง หรือฝาครอบขุ่นจน AI อ่านไม่ได้เลยไม่ว่าจะปรับภาพอย่างไร	ออกแบบปุ่มโหมด "Manual Key-in" บนหน้าจอแอป ให้พนักงานกดเปลี่ยนมาพิมพ์ตัวเลขด้วยมือได้ทันทีเมื่อ OCR ล้มเหลว	
ความเป็นส่วนตัวของข้อมูลผู้เช่า	ประวัติการใช้ไฟอาจสะท้อนพฤติกรรมการอยู่อาศัยหรือช่วงเวลาการเข้า-ออกห้องของผู้เช่า	ใช้ระบบ Login และกำหนดสิทธิ์การเข้าถึงข้อมูลแบบ Role-based Access Control (RBAC) ให้เฉพาะฝ่ายบัญชีและเจ้าของหอพักเข้าดูข้อมูลรายห้องได้	
			
การแจ้งเตือนความผิดปกติ (Alert) บนหน้าจอพนักงานทันที (เช่น กรณีค่าไฟใหม่น้อยกว่าเดือนก่อน หรือหน่วยกระโดดเกินจริง)			
			
รายงานสรุปยอดการใช้ไฟฟ้าประจำเดือนแยกตามห้องและภาพรวมหอพัก			
			
Scope			
			
In Scope			
			
Mobile Application UI สำหรับพนักงาน: หน้าต่างกล้องพร้อมกรอบ Bounding Box และระบบไกด์ไลน์ (Anti-Reflection Indicator) เพื่อช่วยพนักงานเอียงกล้องหลบแสงสะท้อน			
			
ระบบ OCR Processing: โมเดลประมวลผลภาพ (Image Pre-processing) เช่น Perspective Transform ปรับภาพเอียงให้ตรง และแปลงภาพหน้าปัดจานหมุนเป็นตัวเลข 5 หลัก			
			
Local Validation Logic: ระบบคำนวณและแจ้งเตือนความผิดปกติของตัวเลขทันทีที่หน้างานก่อนบันทึก			
			
Offline Data Storage: ระบบบันทึกข้อมูลและรูปภาพลงในเครื่องชั่วคราวเมื่อไม่มีสัญญาณเน็ต และจะซิงค์ขึ้น Cloud อัตโนมัติเมื่อเชื่อมต่อ Wi-Fi (Offline-First)			
			
Web Admin Backend: ระบบหลังบ้านสำหรับจัดการข้อมูลผู้เช่า, ตรวจสอบภาพถ่ายมิเตอร์ย้อนหลัง, และกดออกรายงานสรุปยอดค่าไฟ			
			
Out of Scope			
			
ระบบพิมพ์บิลหรือใบเสร็จรับเงินออกมาเป็นกระดาษโดยตรง (พิมพ์ผ่านระบบภายนอก)			
			
ระบบรับชำระเงินออนไลน์ (Payment Gateway) หรือกระเป๋าเงินดิจิทัล			
			
ระบบแจ้งเตือนบิลค่าไฟไปยังผู้เช่าโดยตรงผ่าน SMS หรือ Line (ส่งมอบเฉพาะข้อมูลสรุปยอดให้ฝ่ายบัญชีไปดำเนินการต่อ)			
			
การเชื่อมต่อกับระบบมิเตอร์อัจฉริยะ (Smart Meter / Digital IoT Meter) ที่ส่งข้อมูลผ่านโครงข่ายไฟฟ้าอัตโนมัติ (โฟกัสเฉพาะมิเตอร์จานหมุนแบบเดิม)			
			
Constraints and Ethics/Privacy			
			
Constraint / Issue	Impact	Response	
ข้อจำกัดทางกายภาพ (แสงสะท้อนและมุมกล้อง)	ภาพสะท้อนจากฝาครอบพลาสติกและมุมที่บิดเบี้ยวทำให้ OCR อ่านตัวเลขผิดพลาด	ออกแบบ UI บังคับให้ถือกล้องเอียง 15-30 องศา และใช้ OpenCV ทำ Perspective Transform ปรับภาพให้ตรงก่อนสแกน	
สภาพแวดล้อมหน้างาน (พื้นที่อับสัญญาณเน็ต)	พนักงานไม่สามารถส่งรูปไปประมวลผลบน Server ระหว่างเดินตรวจตามชั้นได้	พัฒนาสถาปัตยกรรมแบบ Offline-first ให้บันทึกรูปและใช้ Logic ตรวจสอบเบื้องต้นในเครื่องก่อนซิงค์ข้อมูลภายหลัง	
ความก้ำกึ่งของมิเตอร์จานหมุน	ช่วงที่เฟืองตัวเลขกำลังเลื่อนเปลี่ยนหลัก โมเดล AI อาจเกิดความสับสน	กำหนดเป็นเกณฑ์จริยธรรม/สถาปัตยกรรมระบบว่า ต้องให้มนุษย์ (พนักงาน) เป็นผู้ตรวจสอบและกดยืนยันขั้นสุดท้ายเสมอ (Human-in-the-loop) ห้ามระบบเซฟค่าอัตโนมัติโดยไม่มีการตรวจทาน	
ความเสื่อมสภาพของอุปกรณ์หน้างาน	มิเตอร์บางตัวเก่ามาก ตัวเลขจาง หรือฝาครอบขุ่นจน AI อ่านไม่ได้	ออกแบบระบบให้พนักงานสามารถสลับโหมดมาเป็น "พิมพ์ตัวเลขด้วยตนเอง (Manual Key-in)" ได้ทันทีเมื่อ OCR ล้มเหลว	
ความเป็นส่วนตัวและข้อมูลผู้เช่า	ประวัติการใช้ไฟอาจสะท้อนพฤติกรรมการอยู่อาศัยของผู้เช่า	กำหนดสิทธิ์ Role-based Access Control ให้เฉพาะฝ่ายบัญชีและเจ้าของหอพักเท่านั้นที่เข้าดูข้อมูลรายห้องได้
